# Public Demo Notes

Use this note when preparing a public demo of the repository.

## Demo Goals

- Show the app visually.
- Keep build steps simple.
- Explain menu controls clearly.
- Mention fullscreen limitations.
- Keep the local-first message visible.

## Before Sharing

- Check that the README is current.
- Check that screenshots are safe.
- Check that the app builds locally.
- Check that usage notes match the current app behavior.
