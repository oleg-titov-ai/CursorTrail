# Maintenance Notes

This file keeps lightweight maintenance rules for CursorTrail.

## Regular Checks

- Build the app after changing Swift code.
- Keep the menu-bar workflow simple.
- Test trail visibility in normal desktop windows.
- Document fullscreen limitations clearly.
- Keep release notes current.

## Safe Data Rules

Do not commit private screenshots, Apple Developer credentials, certificates, notarization material, personal logs, or unreleased private artifacts.

## Release Hygiene

Before publishing a new update, check build status, Info.plist validity, README accuracy, changelog entries, and usage documentation.
